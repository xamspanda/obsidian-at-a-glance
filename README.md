# At a Glance

At a Glance is a desktop-only Obsidian plugin that renders a recent-work summary through the local Hermes CLI from an `ai-overview` codeblock.

The plugin is intentionally codeblock-driven, like Dataview or Tasks: the markdown source keeps a small control block, and the generated overview is rendered dynamically in reading/preview mode. By default, the overview is not written back into the note source.

## Install

1. Copy this folder to:

   ```text
   /Users/Laura/Obsidian/Primary-Vault/.obsidian/plugins/obsidian-at-a-glance/
   ```

2. Enable **At a Glance** in Obsidian under **Settings -> Community plugins**.

3. Make sure the Obsidian Local REST API is running at:

   ```text
   https://127.0.0.1:27124
   ```

4. Set one of these environment variables before launching Obsidian, or configure the token in the plugin code/settings if needed:

   ```text
   OBSIDIAN_LOCAL_REST_API_TOKEN
   OBSIDIAN_REST_API_TOKEN
   LOCAL_REST_API_TOKEN
   ```

5. Make sure Hermes is installed at:

   ```text
   /Users/Laura/.local/bin/hermes
   ```

## Daily Note Placeholder

Add this block to the daily note template:

````markdown
```ai-overview
```
````

When the daily note renders, the plugin reads recent daily notes directly from the local vault filesystem, then asks Hermes to crawl linked vault context itself. Hermes receives:

- the absolute vault path
- the current note path
- recent daily-note seed paths and contents
- `tasks` codeblock query text from those daily notes
- scanned checkbox task lines from the seed notes
- a hard max crawl-file count
- a priority order for tasks, daily-note links, project/research/idea tags, recently modified linked notes, and journal references
- a stable response template for consistent sections across runs

The generated markdown is intended to render under the user's existing daily-note heading, such as `### At a Glance`. The prompt explicitly tells Hermes not to include an `At a Glance` heading. If Hermes accidentally emits one anyway, the plugin strips it before rendering, including plain leading `At a Glance` text without markdown heading markers.

After a successful generation, the plugin updates the source `ai-overview` codeblock body with the generation timestamp while keeping the rendered overview dynamic:

````markdown
```ai-overview
*Overview generated by Hermes on YYYY-MM-DD HH:MM:SS ZONE*
```
````

The current response template is:

```markdown
**Completed:**
- ...

**Active / overdue tasks:**
- ...

**Upcoming deadlines:**
- ...

**Journal / project highlights:**
- ...

**Newly created / updated notes:**
- ...

**Current goals:**
- ...

*Generated by Hermes · [current time]*
```

## Command Menu

The plugin registers this command:

```text
At a Glance: Regenerate At a Glance for current note
```

It clears the cached generation for the currently open markdown note, immediately changes any currently rendered `ai-overview` block for that note back to a loading state, asks Hermes to regenerate, and then replaces the rendered block in-place when Hermes finishes. You should not need to close/reopen the note or cut/paste the codeblock.

## Settings

Settings are available under **Settings -> Community plugins -> At a Glance**:

- Hermes binary, default `/Users/Laura/.local/bin/hermes`
- Hermes model, default `openai/gpt-5.4-mini`
- Hermes provider, default `openrouter`
- Days to summarize, default `7`
- Maximum crawl files, default `40`
- Log full prompt, default off

The model is pinned at invocation time with:

```text
hermes -m openai/gpt-5.4-mini --provider openrouter -z "<prompt>"
```

This keeps the default Hermes profile in use, preserving normal Hermes/Honcho memory behavior, while still forcing the model/provider for this task.

## Logging

Logs are written as JSON lines to:

```text
/Users/Laura/Obsidian/Primary-Vault/.obsidian/plugins/obsidian-at-a-glance/ai-overview.log
```

By default, the log records metadata, file paths, statuses, and errors. Enable **Log full prompt** only when debugging; prompts may include private journal content.
